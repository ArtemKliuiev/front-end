Маркованні списки

Особливості талиць:

  • розміри таблиці спочатку не встановленні і обчислюються на основі вмісту комірок (загальна ширина визначається автоматично,
    виходячи з сумарної ширини вмісту комірок);
  • максимальна ширина таблиці - це, як правило, весь доступний простір за шириною;
  • вміст комірок за замовчування вирівнюється по центру вертикалі. Якщо вміст комірок різний, то в одній комірці знизу і зверху
    з'явиться пустий простір
Гіперпосилання використовують у трьох випадках:
  • гіперпосилання з якорем для навігації в межах однієї сторінки;
  • відносні адреси, як правило, для навігації в межах одного сайту;
  • абсолютноі адреси, як правило, для переходу на іний сайт.

Структура таблиці в HTML (така ієрархія):
  1. відкрити <table>
  2. додати рядки з допомогою <tr>
  3. додати звичайні комірки через <td> або заголовків через <th> Кількість комірок (тобто елементів <td> і <th>) повинна бути
    однакова в кожному рядку таблиці, тобто всередині кожного тегу <tr>
  4. закрити <table>
